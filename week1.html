День1. Сразу в бой

Зачем нужны теги и как их заводить?

Теги в HTML нужны что бы управлять содержимым и структурой сайта, т.к. браузер не понимает переноса строк списка и тд без т егов.

Теги задаются при помощи спец угловых кавычек.

Синтаксис:

< название тега >

Всякие пояснения по атрибутам

Поясниловка Куча текста:
  1. Парные теги
  2. Не парные теги

Атрибуты тегов

какой то текст поясниловка

Пример использования

<hr width="300">

В примере выше мы показали что линия должна быть шириной в 300 пикселей.

Важно:

Директива DOCTYPE

Директива DOCTYPE используется для того, что-бы определить тип документа и указать браузеру - как обрабатывать нашу страничку. А точнее - какая версия HTML использовалась при ее создании.

Пример использования:

< !DOCTYPE html >

Например, в нашем случае - DOCTYPE указывает браузеру, что страница создается на HTML5 и что ее надо обрабатывать в строгом соответствии этому стандарту.

Комментарии в коде

Комментарии открываются при помощи угловой кавычки и воскл знака с двумя дефисами

Закрываются - двумя дефисами и уловой кавычкой

Пример использования:

< !-- Текст комментария -- >

День 3. Списки и ошибки

Нумерованный список

Назыывается он так потому что все его элементы нумеруются по порядку и могут нумероваться различ переменными

задается через тег <ol>. Все элементы списка добавляются при помощи тега <li>

Пример использования:

<ol>

  1. <li>Элемент нумерованного списка</li>
  2. <li>Элемент нумерованного списка</li>
  3. <li>Элемент нумерованного списка</li>

</ol>

Маркерованный список

Назыывается он так потому что все его элементы помечаются маркерами (могут быть разными)

задается через тег <ul>. Все элементы списка добавляются при помощи тега <li>

Пример использования:

<ul>

  • <li>Элемент маркерованного списка</li>
  • <li>Элемент маркерованного списка</li>
  • <li>Элемент маркерованного списка</li>

</ul>

Важно:

Типы маркеров в обоих списках можно изменить при помощи атрибутов, но в HTML5 это осуждается т.к. тоже самое можно проделать при помощи CSS3. поэтому работу со списками разберем в блоке про CSS.

Список определений

Задаются эти списки при помощи тега <dl>. При чем используется не два тега как в первых списках а сразу три.

Второй тег <dt> - задает термин.

третий тег <dd> - задает определение для термина.

<dl>

<dt>Чекушка</dt>
<dd>Используется многими продвинутыми для быстрого уталения жажды (спрайт - отдыхает)</dd>
<dt>Флян</dt>
<dd>Используется если чекушка не спасла</dd>
<dt>Банка</dt>
<dd>Используется в крайнем случае</dd>

</dl>

Правило - ЧПК :)

ЧПК - это человеко-понятный код.

Что бы вы могли разобраться в коде используйте отступы для вложенных тегов, которые начинаются с новой строки. Отступы задаются при помощи tab

обязательно надо использовать отступы, иначе код будет очень плохо читаем и будет проблемма в нем разобраться, что будет тормозить верстку сайта.

Код без ошибок

Соблюдать вложенность тегов, так как иначе сайт будет не правильно работать, т.е если вы завели один тег внутри второго тега, то этот тег должен закончится внутри этого второго тега

Не правильный пример:

<p> <b> текст </p> </b>

Правильный пример:

<p> <b> текст </b> </p>

Всегда внимательно вводите парные теги и не забывайте их закрывать. Так же, в закрывающем теге должен присутствовать слеш

Всегда будьте придельно внимательны при работе с когод, Т.К. один маленький недостающий символ может испортить весь сайт

почаще сохраняйте код и сразу же обновляйте страницу в браузере и смотрите результат

День 4. Ссылки и картинки

Ссылки на сайте

Ссылки задаются при помощи тега А, который является парным. При чем текст между тегами ссылками называется анкором. А вот сама ссылка задается через атрибут href

Пример использования:

<a href="ссылка"> Анкор ссылки </a>

Ссылка на Мaster-css

Ссылки делятся на 3 вида:

1.Внешние ссылки
Ведут на другой сайт.
2.Внутренние ссылки
они ведут на другую страницу того же самого сайта.
3.Якоря
Они ведут на другой участок той же самой страницы

Переход на главную страницу дневника

Якоря.

Якорь задается при помощи атрибута id. Сначала на странице в нужном месте мы создаем якорь, а потом задаем обычную ссылку только она будет вести на якорь.

Пример использования:

<a id="якорь"> </a>

<a href="#якорь"> Анкор ссылки </a>

Перейти на верх страницы

Атрибуты для ссылок:

href- указывает путь ссылки.

rel="nofollow"- поисковики не учитывают ТИЦ и PR, который может передавать ссылка.

title="подсказка"- всплывающая подсказка при наведении курсора.

target- указывает как должна открываться ссылка. По умолчанию стоит атрибут _self - ссылка открывается в той же вкладке. Если указать атрибут _blank - ссылка откроется в новой вкладке.

Картинки на сайте

Они задаются при помощи тега IMG, который является непарным. Путь к картинке задается через спец атрибут src. Расширения картинок которые мы модем использовать это jpg, png, gif.

Атрибуты для картинок:

alt- - дополнительный специальный атрибут к картинкам. Без него они будут работать, но код будет не валидным и это плохо скажется на оптимизации. Он используется если картинки отключены браузером, тогда вместо них отображается текс из этого атрибута

title="подсказка"- всплывающая подсказка при наведении курсора.

height и width- указывают высоту и ширину нашей картинки. Если указан только один атрибут, то второй подстроится автоматически под пропорции картинки. Могут измерятся в процентах и пикселях.

Важно:

Если указываются значения в проц то проценты вычисляются не от начального размера картинки а от размера блока-родителя.

Пример использования:

<img src="путь к картинке" alt="альтернативный текст">

День 5. Таблицы

Как задаются таблицы?

Таблицы задаются при помощи тега table. В каждой таблице есть строки (тег tr)и в каждой строке содержатся ячейки (тег td).

Атрибуты для ячеек:

colspan - объединение ячеек по горизонтали. Используется только целое число выше 1.

rowspan - объединение ячеек по вертикали. Используется только целое число выше 1.

Дополнительные теги:

caption - этот тег отвечает за заголовок таблицы. Задается только внутри таблицы.

th - отвечает за создание загаловочной ячейки. Текст по умолчанию делается жирным и размещается по центру

col - используется что бы указать доп параметры для колонок таблицы. Этот тег не парный! Испльзуется атрибут width и span.
